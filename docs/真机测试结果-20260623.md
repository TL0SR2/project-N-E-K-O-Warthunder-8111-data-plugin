# 真机测试结果：2026-06-23

## 范围

- 宿主后端、Hosted UI、数据层 `:8112`、War Thunder 真机环境。
- 覆盖 `dry_run=true` 与 `dry_run=false`。
- 同日追加空战 dry_run 监控覆盖 `:8111` 原生遥测、`:8112 /api/telemetry`、插件日志链路；该轮启动中 Hosted UI HTTP 端口未监听，因此只作为运行主链路证据，不重复计入 Hosted UI smoke。
- 本记录只保留聚合结论和安全摘要，不写 raw 玩家名、raw HUD 文本、raw combat.feed 原文或 awards 原文。

## 通过项

- Hosted UI context/action：可返回状态，`set_dry_run` / `test_say` 可调用。
- 数据层连接：插件可持续读取 `:8112 /api/telemetry`。
- 手动身份：`/api/identity` 设置后，数据层返回 `combat.self.source=manual`，并驱动 owned combat attribution。
- 空战数值安全：`spawn`、`low_fuel` warning/critical、`overheat` warning/critical、`stall_risk` warning/critical、`low_alt_danger` warning/critical 均进入 Arbiter / Dispatcher dry_run；重复命中可被 cooldown / scenario gate 解释。
- 空战追加 dry_run：`overspeed` warning/critical、`low_alt_danger` warning、`stall_risk` warning/critical 均被再次观察到；`overspeed` critical 可被 Arbiter `preempt`，重复命中可被 cooldown 解释。
- 空战 owned kill：`combat.feed` 出现 `is_my_kill=true`，证明 air ownership feed 可用。
- 空战 owned death：手动 identity 提前设置后，新 `combat.feed` 出现 `is_my_death=true`；追加监控中坠毁类 feed 也能生成 `you_died`，Arbiter allowed/preempt，Dispatcher dry_run 输出。
- 陆战 owned kill：`combat.feed` 出现多条 `is_my_kill=true`；插件生成 `you_killed`，Arbiter allowed，Dispatcher 输出。
- 陆战 owned death：`combat.feed` 出现 `is_my_death=true`；插件生成 `you_died`，Arbiter allowed，Dispatcher 输出。
- `dry_run=false`：`test_say`、`you_killed`、`you_died` 均被 `push_message` 接受；现场确认猫娘实际开口。
- T-Observe：`observe.last_event` / `last_decision` / `last_output_status` 能解释 allow、dry_run、pushed。
