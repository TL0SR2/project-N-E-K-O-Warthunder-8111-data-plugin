## 优先（DTO 冻结前必须）

- [x] **T-Safety: output text sanitizer** ✅ 已完成
  - 目标：防止猫娘复读不良玩家 ID、`hudmsg`、`combat.feed` 原文，避免辱骂、涉政、擦边、仇恨、广告、联系方式、奇怪符号或 prompt injection 文本进入猫娘输出。
  - 放置位置：`neko_warthunder` 的 `NekoDispatcher` / prompt builder 前。数据层继续保留 raw fact，不负责输出安全；Detector / Scenario / Arbiter 不承担文本过滤职责。
  - 关键策略：
    - raw 玩家名 / hudmsg / combat.feed 原文只进 audit/debug。
    - safe 字段才允许进入 prompt。
    - 默认 generic 文案，不朗读陌生玩家名（例如"一名敌人"、"对手"、"某位玩家"）。
    - `hudmsg` / `combat.feed` 原文默认不进 prompt。
    - 不确定时宁可不读原文。
  - 当前状态：已新增 `adapters/text_safety.py`，并在 `adapters/neko_dispatcher.py` 的 prompt builder 前接入。
  - 验收：sanitizer 单测、dispatcher prompt 测试、`push_message.parts[].text` 不包含 unsafe raw 的合同测试已覆盖；hudmsg / combat.feed / awards 常见自由文本字段族已默认 blocked；逻辑自检当前基线 `158/158 passed`。
  - 后续关系：kill / death generic 输出已通过真机 dry_run 与 `dry_run=false` push smoke；hudmsg / awards / 其他 free-text 正式播报仍需真机 dry_run 安全验证。数值安全事件不被 T-Safety 阻塞。

- [x] **补 overspeed（超速）告警 flag** ✅ 已完成（v1.6 合并）
  - 做法：`vehicle_profiles.json` 加 `overspeed_warn_kmh` / `overspeed_critical_kmh`（逐类 never-exceed，
    喷气/喷气轰另给选填 `overspeed_warn_mach` / `overspeed_critical_mach`，IAS 与马赫判据取或）；
    `wt_processor.py` 加 `_process_overspeed`，产出 `overspeed_warn` / `overspeed_critical` 两级 flag。
  - 注意：① **不受起落架抑制**（放襟翼/起落架时更易撕裂，反而更危险）；② 阈值是按类别的**工程估值**，
    游戏不存静态限速、各机撕裂线差异大，**尚无真机校准**，需后续按机型微调（同 stall/aoa 性质）。
  - 验收：`/api/processed.flags` 已出现 `overspeed_warn` / `overspeed_critical`，见接口文档 4.5 告警代码表。
