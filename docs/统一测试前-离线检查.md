## 2.1 宿主运行时注册注意

当前工作区里 `N.E.K.O\plugin\plugins\neko_warthunder` 是指向独立插件仓库的 junction。手动启动宿主时，如果未设置插件扫描根，宿主可能把真实目标路径判定为插件根之外。建议使用：

```powershell
cd D:\Users\zheng\Documents\Code\N-E-K-O-Warthunder\N.E.K.O
$env:PLUGIN_CONFIG_ROOT = "D:\Users\zheng\Documents\Code\N-E-K-O-Warthunder"
uv run python launcher.py
```

启动后确认注册：

```powershell
Invoke-RestMethod http://127.0.0.1:48916/plugins
Invoke-RestMethod -Method Post http://127.0.0.1:48916/plugins/refresh
Invoke-RestMethod -Method Post http://127.0.0.1:48916/plugin/neko_warthunder/start
```

若 refresh 报出外层独立仓库目录名的重复/无效项，但 `neko_warthunder` 通过 junction 路径可启动、Hosted UI context 返回状态，则可忽略该重复项。

```powershell

```

该清单会把 P1/P2 项展开成“操作 / 监控 / 通过 / 失败 / 数据层缺口”。

真机测试进行中需要快速看当前链路状态时：

```powershell
uv run python tools\live_monitor.py
uv run python tools\live_monitor.py --json
```

该工具只做只读检查：health、Hosted UI context、`/api/telemetry` 聚合计数、free-text dry_run-only 状态、replay 降级状态、T-Observe last decision/output 和最近日志异常计数。输出不得包含 raw 玩家名、raw HUD 文本、raw combat.feed 或 awards 原文。若现场出现自由文本源，应看到 `free_text=dry_run_only(...)`、`FreeText detail: awards=.../blocked` 等逐源摘要；若现场出现 `replay=true`，应看到 `replay=suppressed(detector_suppressed/replay)` 和 `output_blocked=True`。
