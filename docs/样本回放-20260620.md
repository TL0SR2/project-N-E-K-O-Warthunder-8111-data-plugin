## 对下一轮真机测试的影响

下次真机测试应优先补这些缺口；具体现场步骤可直接由 `tools/live_test_plan.py` 展开为“操作 / 监控 / 通过 / 失败 / 数据层缺口”清单：

1. 回放降级：确认 `/api/telemetry` 出现 `replay: true` 时插件静默 Detector 输出。
2. 超速 critical：触发或采集 `overspeed_critical`，验证优先级和 Arbiter 抢占语义。
3. kill/death ownership：通过 Hosted UI 设置玩家名后，确认 `combat.feed[].is_my_kill` / `combat.feed[].is_my_death` 出现并驱动 dry_run。
4. 油温 notice：采集 `hud_notices.feed[].code=oil_overheat`，确认映射到现有 `overheat`，且 raw 文本不进 prompt。
5. 动力故障 notice：采集 `powertrain_failure` 或等价字段后，再决定是否新增事件或继续不播。
6. notice severity：确认 warning / critical 档位是否稳定，不稳定则继续只用 safe code + 插件侧默认等级。

## 结论

这包样本可以支撑离线回归，但不能替代下一轮真机 dry_run。当前可以认为：

- 数值安全主链路离线回归可用。
- `/api/identity` manual 帧存在，适合继续做身份接缝回归。
- kill/death ownership、replay true、overspeed critical、oil overheat、powertrain failure 仍需要真机或新样本补齐。
- 自由文本相关路径继续受 T-Safety 约束，正式播报前必须先通过 dry_run。
